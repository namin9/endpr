# 비밀번호 재설정 링크 운영 가이드

## 개요
슈퍼 관리자는 특정 사용자에 대해 **1회용 비밀번호 재설정 링크**를 발급합니다. 사용자는 해당 링크에서 비밀번호를 변경하며, 토큰은 즉시 폐기됩니다.

## 발급 절차
1. 슈퍼 관리자가 `POST /cms/users/:id/password-reset-link` 호출
   - 요청 바디: `tenant_id`, (선택) `expires_in_seconds`
2. 응답으로 `url` 과 `expires_at` 수신
3. 사용자가 링크에 접속하여 비밀번호 변경

## 만료 및 재발급
- 토큰은 기본 1시간 후 만료됩니다.
- 만료된 링크는 `410 Invalid or expired token` 으로 처리됩니다.
- 재발급이 필요한 경우 동일 사용자에 대해 새 링크를 다시 생성합니다.

## 보안 권장 사항
- 링크는 안전한 채널(보안 메신저/암호화 이메일)로 전달합니다.
- 발급 후 일정 시간이 지나면 새 링크로 재발급하는 것을 권장합니다.
- 발급 시점에 사용자에게 비밀번호 변경 완료 여부를 확인합니다.

## 관련 엔드포인트
- `POST /cms/users/:id/password-reset-link`
- `POST /cms/auth/password-reset`
